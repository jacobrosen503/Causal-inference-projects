# Data Dictionary

## `city_month_panel` (primary analysis file)

Panel grain: city x month.

| Column | Type | Description |
|--------|------|-------------|
| `city` | string | City name |
| `country` | string | Country |
| `month` | date | Month start date |
| `year_month` | string | `YYYY-MM` |
| `month_idx` | int | 0-based month index within city |
| `is_treated_city` | bool | True for NYC and Florence |
| `post` | bool | True for months at or after enforcement date |
| `treat_col` | string | Treatment indicator name (`treat_nyc`, `treat_florence`, or empty) |
| `log_listings` | float | Log count of active listings in that city-month |
| `listings` | int | Raw listing count |
| `mean_price_usd` | float | Mean (or median) nightly price in USD |
| `log_price` | float | Log of mean_price_usd |
| `availability_rate` | float | Mean availability_365 / 365 across listings |
| `entire_home_share` | float | Share of listings that are entire home/apt |
| `is_simulated` | bool | True if generated by simulate_panel.py; False if built from real data |

## `regulations.csv`

| Column | Description |
|--------|-------------|
| `city` | City name |
| `country` | Country |
| `policy` | Policy description |
| `enforcement_date` | Date regulation took effect (YYYY-MM-DD) |
| `treat_col` | Corresponding indicator column name |
| `notes` | Context and caveats |

## Primary outcomes

| Outcome | Interpretation | Expected direction (NYC LL18) |
|---------|---------------|-------------------------------|
| `log_listings` | Listing supply | Strong negative (-1.6 to -1.8) |
| `mean_price_usd` | Nightly price | Positive (supply reduction) |
| `availability_rate` | Share of days available | Negative |
| `entire_home_share` | Share of entire-home listings | Negative (hosts shift to private room) |

## Inside Airbnb raw data (after download)

Raw files land in `data/raw/{city}/` as `{city}_{YYYY-MM-DD}_listings.csv.gz`.

Key columns used from raw listings:
- `price`: nightly price (string, e.g. "$150.00")
- `room_type`: "Entire home/apt", "Private room", "Shared room", "Hotel room"
- `availability_365`: days available in next 365 days
- `availability_30`: days available in next 30 days (fallback)
